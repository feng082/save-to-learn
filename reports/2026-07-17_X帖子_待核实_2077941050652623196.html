一句话结论

这条帖子的真正价值，不是推荐某一个生图模型，而是把“做一张封面”拆成了内容理解 → 视觉决策 → 提示词生成 → 生图执行的可复用工作流。

作者在做什么

作者宣称开源一个面向小红书封面制作的 Skill，目标用户是不懂设计、也不想手写 prompt 的创作者。

文章丢进去，三轮问答，直接吐出能跑图的封面提示词；固定 3:4 画幅，同时加入安全区、人物参考和素材参考的多参考图写法。

首次配置需要保存一张正脸参考图并选择生图模型；之后由 Agent 自动提炼文章内容并生成封面方案。

8 步工作流

01

安装与首次配置

保存自己的正脸参考图，确认后续使用的生图模型。

02

输入文章

把文章交给 Agent，让它自动阅读、提炼和理解主题。

03

第一轮：构图与标题

根据内容推荐 10 种构图风格，并自动生成候选标题。

04

第二轮：人物与素材

补充人脸参考图、UI 截图、产品图等视觉素材。

05

第三轮：视觉细节

确定表情、背景色调、字体和字色；不想选择时可交给模型。

06

输出完整提示词

包含 3:4 构图、安全区，以及“图 1 人脸 + 图 2 素材”的多参考图写法。

07

交给生图工具

可尝试即梦、Nano Banana 或 GPT-Image；配置好 API 时也可以直接在对话中生成。

08

校对与局部重生成

检查标题错字，不满意哪一项就只改哪一项重新生成。

值得复用的设计原则

FOR SKILL DESIGN
  • 把开放式创作任务改造成有限轮次问答。
  • 每一轮只解决一组决策，降低认知负担。
  • 将画幅、安全区、参考图关系写进固定模板。
  • 用真实案例构成示例库，给模型稳定参照。
  • 保留局部修改机制，避免每次从头生成。
FOR CONTENT
  • 开头先承诺用户能感知的结果。
  • 用编号流程把复杂工具变成可执行步骤。
  • 用“人脸一致性”“标题错字”等具体痛点沟通。
  • 同时说明适用工具、门槛和注意事项。
  • 把“爆款感”拆成可观察、可复用的视觉结构。

可复用工作流模板

输入：一篇文章 + 人物参考图 + 产品/UI/案例素材

第一轮：提炼主题，给出 3–10 种构图方向和标题候选
第二轮：确认人物、产品、界面等参考素材及优先级
第三轮：确认表情、背景、字体、字色和信息层级

输出：固定画幅 + 安全区 + 主标题 + 辅助信息
      + 多参考图关系 + 生图提示词
复核：检查文字、主体一致性和平台裁切风险
迭代：只重生成不满意的模块

作者观点与事实边界

作者强调
  • 生图模型最好支持多参考图，否则人脸一致性难以保持。
  • Skill 内置 8 组按真实爆款封面逆向还原的示例提示词。
  • 纯提示词生成，不用 API Key，支持 Codex、Claude Code 等 Agent。
需要验证
  • “爆款”没有附带可核验的数据或转化结果。
  • 零环境依赖不代表生图平台本身不需要账号或配置。
  • 人脸一致性最终取决于模型的多参考图能力。
  • 帖子正文没有直接给出仓库地址。

延伸问题

  1. 如何把 10 种封面构图整理成可复用的 STYLE 体系？
  2. 人物图和产品图的参考权重应该怎样写进提示词？
  3. 如何建立标题错字与可读性检查？
  4. 3:4 封面的安全区如何适配不同设备裁切？
  5. 如何用同一套 Skill 批量生成 A/B 测试封面？
  6. “爆款封面”应该由哪些可量化指标定义？
  7. 如何把账号的字体、色彩和人物形象写进 Skill 的长期记忆？
  8. 如何让 Agent 根据文章类型自动选择构图模板？
  9. 如何在不泄露真人隐私的前提下管理人脸参考图？
  10. 如何接入发布前的标题、版权和敏感词检查？

我的想法

最值得学习的不是某个具体模型，而是“固定输入、分轮决策、结构化输出、局部迭代”的工作流。它把设计能力从一次性灵感，变成可以持续积累和复用的系统。

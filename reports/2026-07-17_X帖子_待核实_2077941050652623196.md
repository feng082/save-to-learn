## 一句话概括

作者开源一个面向小红书封面制作的 Skill：把文章交给 Agent，经过三轮问答，生成固定 3:4 画幅、包含安全区和多参考图写法的封面提示词，再交给生图模型执行。

## 内容摘要

帖子宣称该 Skill 面向不懂设计、也不想手写 prompt 的用户。首次配置时保存一张自己的正脸照并选择生图模型；之后输入文章，Agent 自动提炼内容，依次完成构图风格与候选标题、人物与素材参考、表情与视觉细节选择，最后输出可直接用于生图的完整提示词。

作者特别强调两点：生图模型应支持多参考图，否则人物一致性难以保持；Skill 内置 8 组根据真实爆款封面逆向还原的示例提示词，用于提升输出的具体程度。作者还声称该 Skill 只生成提示词、不依赖 API Key、没有额外环境依赖，可供 Claude Code、Codex 等支持自定义 Skill 的 Agent 使用。

## 工作流拆解

1. 安装 Skill，并保存一张正脸参考图，确认生图模型。
2. 将文章内容交给 Agent，自动阅读和提炼。
3. 第一轮问答：根据文章推荐 10 种构图风格，并生成候选标题。
4. 第二轮问答：补充人脸参考图、UI 截图、产品图等素材。
5. 第三轮问答：确定表情、背景色调、字体和字色；用户不选择时可交给模型决定。
6. 输出完整提示词，包含 3:4 构图、安全区，以及“图 1 人脸 + 图 2 素材”的多参考图写法。
7. 将提示词交给即梦、Nano Banana 或 GPT-Image 等生图工具执行。
8. 核对标题错字；不满意时只修改对应部分并重新生成。

## 10 个核心观点

1. 封面制作可以被拆成“内容理解—视觉决策—提示词生成—生图执行”四个阶段。
2. Skill 的主要价值不是直接画图，而是把设计决策流程结构化。
3. 三轮问答把一次性复杂需求拆成逐步收敛的选择过程。
4. 第一轮先处理构图风格和标题，优先解决封面的信息架构。
5. 第二轮加入人物和产品等参考素材，解决主体与内容的对应关系。
6. 第三轮集中处理表情、色调、字体和字色，减少来回修改成本。
7. 固定 3:4 画幅和安全区，使结果更贴近小红书封面发布场景。
8. 多参考图写法是保持人物、产品和界面关系稳定的关键提示词组件。
9. 示例提示词相当于隐性的视觉规范库，能把抽象的“爆款感”转成可模仿的结构。
10. 最终保留人工校对和局部重生成，尤其要检查封面标题的错字。

## 论证结构

帖子采用“结果承诺 → 流程展示 → 技术提醒 → 使用门槛说明”的结构：先用“文章丢进去、三轮问答、直接得到能跑图的提示词”说明结果；再用 8 步流程证明这不是一句空泛的工具宣传；接着指出多参考图模型和示例提示词两个关键条件；最后用“不用 API Key、零环境依赖”降低尝试门槛。

## 值得学习的地方

### 对 Skill 设计

- 将开放式创作任务改造成有限轮次的问答流程。
- 每轮只解决一组决策，避免用户一开始面对过多参数。
- 把画幅、安全区、参考图关系等容易遗漏的约束写入固定输出模板。
- 用真实案例逆向形成示例库，让模型有稳定的视觉参照。
- 保留“只改不满意的部分”的迭代机制，避免每次从头生成。

### 对内容表达

- 开头直接给出用户能感知的结果，而不是先解释技术原理。
- 使用编号流程把复杂工具包装成可执行步骤。
- 用“人脸一致性”“标题错字”等具体痛点替代抽象的效率宣传。
- 同时说明适用工具、运行门槛和注意事项，降低读者试用成本。

## 可复用的方法模板

```text
输入：一篇文章 + 人物参考图 + 产品/UI/案例素材

第一轮：提炼主题，给出 3–10 种构图方向和标题候选
第二轮：确认人物、产品、界面等参考素材及其优先级
第三轮：确认表情、背景、字体、字色和信息层级

输出：固定画幅 + 安全区 + 主标题 + 辅助信息 + 多参考图关系 + 生图提示词
复核：检查文字、主体一致性和平台裁切风险
迭代：只重生成不满意的模块
```

## 实际应用建议

- 用于把长文章快速转成小红书封面草案。
- 用于个人 IP 账号建立固定的人物形象和封面规范。
- 用于产品发布、课程宣传、案例复盘和工具教程的封面批量试稿。
- 将“10 种构图风格”替换成自己的风格库，形成账号专属模板。
- 将字体、色彩和安全区参数固定为品牌规范，进一步减少选择。

## 值得质疑的地方

- “爆款”属于作者的宣传判断，帖子本身没有给出可核验的样本数据或转化数据。
- “零环境依赖”描述的是提示词生成流程；实际生图仍取决于所使用平台、模型和账号配置。
- “人脸保持一致”高度依赖模型的多参考图能力，不能仅凭 Skill 保证。
- “仓库地址评论区可见”在当前读取到的正文中没有提供具体仓库 URL。
- 配图的视觉内容尚未独立下载核验，因此不能据此判断示例提示词的实际质量。

## 我的想法

这条帖子的真正价值在于把“做一张封面”改写成一个可复用的决策系统。它没有试图让模型一次性猜完所有设计细节，而是把内容、素材和视觉参数分轮确认，再输出可执行的提示词。对个人创作者而言，最值得复用的不是某个具体模型，而是“固定输入、分轮决策、结构化输出、局部迭代”的工作流。
